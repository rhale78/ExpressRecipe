### Test Entities Created

#### 1. Product.cs
```csharp
[Table("Products")]
[Cache(CacheStrategy.Memory, MaxSize = 1000, ExpirationSeconds = 300)]
public partial class Product
{
    // Id property auto-generated by framework
    public string Name { get; set; } = string.Empty;
    public decimal Price { get; set; }
    public int StockQuantity { get; set; }
    public string Category { get; set; } = string.Empty;
    public DateTime CreatedDate { get; set; }
    public bool IsActive { get; set; } = true;
}
```

**Demonstrates**:
- Basic `[Table]` attribute usage
- Memory caching configuration
- `partial` keyword for source generation
- Framework auto-generates Id property

#### 2. Order.cs
```csharp
[Table("Orders")]
[Cache(CacheStrategy.Memory, ExpirationSeconds = 600)]
[SoftDelete]
[AutoAudit]
public partial class Order
{
    // Framework auto-generates: Id, CreatedBy, CreatedDate, ModifiedBy, ModifiedDate,
    // IsDeleted, DeletedDate, DeletedBy
    public string OrderNumber { get; set; } = string.Empty;
    public int CustomerId { get; set; }
    public decimal TotalAmount { get; set; }
    public string Status { get; set; } = "Pending";
    public DateTime OrderDate { get; set; }
}
```

**Demonstrates**:
- Soft delete functionality
- Auto-audit field generation
- Combined attributes working together

#### 3. Customer.cs
```csharp
[Table("Customers")]
[Cache(CacheStrategy.Memory, ExpirationSeconds = 900)]
[StagingTable(SyncIntervalSeconds = 30)]
[AutoAudit]
public partial class Customer
{
    // Framework auto-generates: Id, CreatedBy, CreatedDate, ModifiedBy, ModifiedDate
    public string FirstName { get; set; } = string.Empty;
    public string LastName { get; set; } = string.Empty;
    public string Email { get; set; } = string.Empty;
    public string Phone { get; set; } = string.Empty;
    public bool IsActive { get; set; } = true;
}
```

**Demonstrates**:
- Staging table for high-write scenarios
- Auto-audit with staging table
- Background synchronization configuration

## Technical Challenges Encountered

### 1. Source Generator Configuration

**Issue**: Source generator not running initially

**Solution**: Configure as Analyzer with `OutputItemType="Analyzer"` and `ReferenceOutputAssembly="false"`

### 2. Attribute Namespaces

**Issue**: `CacheAttribute`, `TableAttribute` not found

**Solution**: Use `HighSpeedDAL.Core.Attributes` namespace

### 3. DataManagement Attributes

**Issue**: `SoftDeleteAttribute`, `AutoAuditAttribute` not found

**Solution**: Add HighSpeedDAL.DataManagement reference

### 4. DatabaseConnectionBase Setup

**Issue**: TestDatabaseConnection implementation unclear

**Solution**: Follow SimpleCrudExample pattern, override `Provider` and `GetConnectionStringKey()`

### 5. DAL Constructor Parameters

**Issue**: ProductDal requires ILogger, IDbConnectionFactory, RetryPolicyFactory

**Solution**: Pending - need to simplify for test scenarios

## Lessons Learned

1. **Source Generator Setup**: Must be configured as Analyzer, not regular reference
2. **Namespace Awareness**: Attributes spread across Core and DataManagement
3. **Connection Pattern**: DatabaseConnectionBase requires specific implementation
4. **Test Complexity**: Real framework usage tests more complex than component tests
5. **DI Requirements**: Framework expects full DI setup even in tests
