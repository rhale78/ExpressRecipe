# HighSpeedDAL Proper Integration Guide

## Understanding HighSpeedDAL Framework

HighSpeedDAL is an **attribute-driven, source-generator-based** Data Access Layer framework. The key insight is that **DAL classes are AUTO-GENERATED by Roslyn source generators**, not manually written.

### Core Concepts

1. **Attribute-Driven**: Entities are decorated with attributes (`[Table]`, `[Cache]`, `[DalEntity]`, etc.)
2. **Source Generator**: Roslyn analyzers generate DAL classes at compile time
3. **Zero Manual SQL**: All CRUD operations generated automatically
4. **Convention Over Configuration**: Smart defaults minimize configuration

### Example from HighSpeedDAL Repository

#### Entity Definition
```csharp
using HighSpeedDAL.Core.Attributes;

[Table("Users")]
[Cache(CacheStrategy.Memory, MaxSize = 1000, ExpirationSeconds = 300)]
[StagingTable(SyncIntervalSeconds = 30)]
[DalEntity]  // KEY: Triggers source generator to create UserDal class
public partial class User
{
    public int Id { get; set; }
    public string Username { get; set; } = string.Empty;
    public string Email { get; set; } = string.Empty;
    public DateTime CreatedAt { get; set; }
    public bool IsActive { get; set; } = true;
}
```

#### Generated DAL Class (Auto-Generated by Source Generator)
```csharp
// UserDal.cs - AUTO-GENERATED, NOT MANUALLY WRITTEN
public class UserDal
{
    public async Task<User> InsertAsync(User entity) { /* generated */ }
    public async Task<User?> GetByIdAsync(int id) { /* generated */ }
    public async Task<List<User>> GetAllAsync() { /* generated */ }
    public async Task UpdateAsync(User entity) { /* generated */ }
    public async Task DeleteAsync(int id) { /* generated */ }
    public async Task BulkInsertAsync(IEnumerable<User> entities) { /* generated */ }
    // ... all CRUD methods auto-generated
}
```

#### Usage
```csharp
// Register auto-generated DAL
services.AddSingleton<UserDal>();

// Use it
var userDal = serviceProvider.GetRequiredService<UserDal>();
var user = await userDal.GetByIdAsync(1);
await userDal.UpdateAsync(user);
```

## Current ExpressRecipe Implementation

Since HighSpeedDAL NuGet packages require GitHub authentication and source generators, we've implemented a **HighSpeedDAL-inspired pattern** that follows the same principles:

### What We've Implemented

1. ✅ **Generic Base Class Methods** - SQL auto-generation in `DalOperationsBase`
2. ✅ **Minimal DAL Classes** - Thin wrappers around base methods
3. ✅ **Table Name + Mapper Pattern** - Convention over configuration
4. ✅ **Zero SQL in Derived Classes** - All SQL in base class

### How We Adapted the Pattern

```csharp
// Base class with generic SQL auto-generation
public abstract class DalOperationsBase<TEntity, TConnection>
{
    protected Task<TEntity?> GetByIdGenericAsync<TKey>(string tableName, TKey id, ...)
    protected Task<List<TEntity>> GetAllGenericAsync(string tableName, ...)
    protected Task<int> InsertGenericAsync(string tableName, object entity, ...)
    protected Task<int> UpdateGenericAsync(string tableName, object entity, ...)
    protected Task<bool> SoftDeleteGenericAsync<TKey>(string tableName, TKey id, ...)
}

// Minimal DAL class - delegates everything to base
public class ProductDal : DalOperationsBase<ProductDto, ProductConnection>
{
    private const string TableName = "Product";
    
    public Task<ProductDto?> GetByIdAsync(Guid id) 
        => GetByIdGenericAsync(TableName, id, MapFromReader, ct);
    
    public Task<List<ProductDto>> GetAllAsync() 
        => GetAllGenericAsync(TableName, MapFromReader, ct);
    
    // Only 121 lines total - mimics auto-generated pattern
}
```

## Migration to Full HighSpeedDAL

To migrate to the actual HighSpeedDAL framework:

### Step 1: Add NuGet Packages
```bash
# Add GitHub Packages source (requires PAT)
dotnet nuget add source https://nuget.pkg.github.com/rhale78/index.json \
  --name github \
  --username YOUR_GITHUB_USERNAME \
  --password YOUR_GITHUB_PAT

# Install packages
dotnet add package HighSpeedDAL.Core
dotnet add package HighSpeedDAL.SqlServer
dotnet add package HighSpeedDAL.SourceGenerators
```

### Step 2: Update Entities
```csharp
using HighSpeedDAL.Core.Attributes;

[Table("Product")]
[Cache(CacheStrategy.TwoLayer, MaxSize = 10000, ExpirationSeconds = 900)]
[DalEntity]  // Triggers source generator
public partial class ProductEntity  // Must be partial for generator
{
    public Guid Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public string? Brand { get; set; }
    // ... rest of properties
}
```

### Step 3: Remove Manual DAL Classes
Delete `ProductDal.cs` and `IngredientDal.cs` - source generator creates them automatically.

### Step 4: Update Service Registration
```csharp
// Register connection
services.AddSingleton<ProductDatabaseConnection>();

// Register auto-generated DAL (ProductEntityDal created by generator)
services.AddSingleton<ProductEntityDal>();
```

### Step 5: Update Service References
```csharp
// Old: IProductDal
// New: ProductEntityDal (auto-generated class name)
public class ProductService
{
    private readonly ProductEntityDal _productDal;
    
    public ProductService(ProductEntityDal productDal)
    {
        _productDal = productDal;
    }
}
```

## Benefits of Full HighSpeedDAL

1. **Zero Manual DAL Code** - Source generators create everything
2. **Type Safety** - Compile-time errors if entity changes
3. **Consistent API** - All DALs have same methods
4. **Auto-Updates** - DAL regenerates when entity changes
5. **Advanced Features** - Staging tables, memory-mapped files, etc.
